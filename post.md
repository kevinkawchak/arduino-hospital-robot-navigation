## [LinkedIn 29Jan26 Post](https://www.linkedin.com/posts/kevin-kawchak-38b52a4a_realrobots-activity-7422843903894212608-F2vR)

Accessing physical robots remotely with custom instruction implementation is a useful approach to test code prior to scaling operations. Here, an Arduino robot in Bilbao, Spain was accessed from San Diego, CA in order to test a hospital_robot_navigation.ino script generated by Opus 4.5 Extended. (1, 2) Full 2:27 Video Download: https://lnkd.in/gXsUyUT8, RL Video: https://lnkd.in/g2WqU-Cu

Main Limitations Included:
Robot lag over the internet, noticeable when using manual navigation
Simple hospital medication delivery script, with no reinforcement learning

Robot Instructions:
NURSE_STATION - Initial pickup point
TO_PATIENT_1 / AT_PATIENT_1 - First delivery
TO_PATIENT_2 / AT_PATIENT_2 - Second delivery
TO_PHARMACY / AT_PHARMACY - Prescription pickup
RETURN_TO_NURSE - Safe return path
MISSION_COMPLETE - End state

Opus Prompt:
“For the LandLab Arduino robot (code) screen shot attached, provide”… “A custom course of your chosing that a hospital robot would have to follow”… “Be sure to provide a text description regarding what the robot is expected to do in each case, and what all is required to input after pushing the orange "Access" button.”

Results: Console: “HOSPITAL ROUNDS COMPLETE” “All deliveries successful!”. The robot made additional turns as the experiment progressed. However, the robot was navigating an unstructured environment, without appropropriate markers.

Future studies will likely involve aquiring larger bipedal robots for intelligence and locomotion/manipulation R&D.

UPDATE 29Jan26 RL Code update (Q-learning) by Opus; Episode 1: Completed, Episode 2: ~78% (time limit). Episode 2 reward was 59% higher at same point (118 vs. 74). However, 15% accuracy. See GitHub for run file.
